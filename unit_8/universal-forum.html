Universal Forum

Employer: Big Fish Games

Due to NDA compliance, key information regarding Big Fish Games will be omitted.

The Universal Forum project is intended to be a universal layout for our internal game studio to use when they wanted to create a forum for one of their games. The forum itself would not be created in house, as we worked with a 3rd party to manage our forums.

Tech limitations

Like all projects, it's always a great start to identify if you have any limitations. This project did have it's limitations.

First limitation, I have to use the 3rd party that Big Fish Games already paid for the forum service. This service was not going to be creating any new functionality for their forum service, as we were needed to select from their existing forum options.

Second limitation, the new forum design must be compatible with an existing forum being used by an existing game.

Forum options

After learning what my limitations were, I wanted to reach out to the studio team figure out what type of forum they believed could be useful for them and their game. I also researched some existing forums that the 3rd party service we're using has done.

Once I gathered some nice options for look and feel of the forums, I started to review the 3rd party's "Addons". These Addons are the 3rd party's forum options. While reviewing these addons, I noticed that they were a mix of what a user would see, what a moderator would see, and options for backend reporting.

Considering that I would need to maintain consideration for our existing forum for Dungeon Boss, I reached out to the moderators to see which addons they use and how they felt about them.

With the idea of supporting future games, I identified some of the producers for these games and presented them a card sort hosted by Optimal Workshop to identify which forum addons they would find useful.

Next Step in Universalizing

My next step to making the forums universal is creating a standard on discussion categories. The reason is because when viewing what Dungeon Boss did, the categories were very game specific.

So I took note of every category and created an open card sort using Optimal Workshop. I took this sorting exercise and shared it internally with the employees.

Based off the results, I created a first click test using Optimal Workshop to see if the users could take common tasks and identify where to find the appropriate category.

Apply to designs

Now I take my data and create a wireframes

MISCOMMUNICATION

Once I've finalized my wireframes and send them to the 3rd party, I learn that they don't "design" and everything must sumbitted in PSD files, and pixel perfect.

MISCOMMUNICATION #2

The 3rd party also requires a default view of the forum that will be intially stamped first before a game producer could change the values to personalize the forum to their game.

Results

The designs were submitted. Unofortunately the submitted changes were never carried out completetly for the existing game, and none of the new games utilized the forum services.
